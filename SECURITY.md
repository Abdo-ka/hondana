# Security Policy

## Supported versions

Hondana is in early development. Security fixes are applied to the latest release
and the `main` branch only.

| Version | Supported |
| ------- | --------- |
| latest `0.x` | ✅ |
| older `0.x` | ❌ |

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, use one of these private channels:

- Open a **private security advisory** via GitHub:
  **Security → Advisories → Report a vulnerability** on the repository, or
- Contact the maintainer **[@Abdo-ka](https://github.com/Abdo-ka)** on GitHub.

Please include:

- A description of the vulnerability and its impact.
- Steps to reproduce (a proof of concept if possible).
- The affected version / commit, and your platform (iOS/Android).

## What to expect

- We aim to acknowledge your report within **72 hours**.
- We'll investigate, keep you updated on progress, and credit you (if you wish)
  once a fix is released.
- Please give us a reasonable window to release a fix before any public
  disclosure.

Thank you for helping keep Hondana and its users safe.
